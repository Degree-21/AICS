# CARE 规格框架

概述
- CARE 将关键规格要素映射为 AI 友好的执行结构

组成
- C — Context（上下文）：背景、目的、假设
- A — Action（行动）：功能需求与少样本示例
- R — Result（结果）：输出与可测试的验收标准
- E — Evaluation（评估）：非功能需求与各类约束

使用建议
- 每份规格都应包含 CARE 四要素
- 示例需具体，验收标准需可度量
- 非功能需求需显式声明（安全、性能、可靠性）

检查清单
- [ ] 上下文完整
- [ ] 功能需求明确
- [ ] 边界情况已列出
- [ ] 示例已提供
- [ ] 验收标准可度量
- [ ] 非功能需求显式
