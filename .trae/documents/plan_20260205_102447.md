# 重构安装与使用流程计划

## 1. 修正安装指南
目前的安装指南中建议用户克隆后运行脚本将文档“安装”回自身目录 (`aics-docs/scripts/init-docs.sh ./aics-docs`)，这会导致文件冲突和逻辑冗余。
我将简化安装流程，**只保留 `git clone` 或 `git submodule`**，删除多余的脚本执行步骤。脚本 `init-docs.sh` 仅保留为“将文档提取到其他自定义目录”的高级用法。

**修改文件：**
- `README.md`
- `README.zh.md`
- `README.en.md`
- `docs/INSTALL.zh.md`
- `docs/INSTALL.en.md`
- `docs/USAGE.zh.md`
- `docs/USAGE.en.md`

## 2. 完善 Step 0：项目初始化
在 Step 0 中，明确让 AI 生成 `project-context.md`，并处理“空项目”的情况（推荐技术栈并生成脚手架）。

**修改内容：**
- 确保提示词明确指出将上下文保存到 `aics-docs/project-context.md`。
- 增加对 `.gitignore` 的说明（建议忽略 `project-context.md`，避免污染文档库更新）。

## 3. 统一对话剧本
确保所有文档中的对话剧本（README 和 USAGE）都遵循新的 **Step 0 -> A -> B -> C -> D** 流程。

## 执行步骤
1.  **更新 INSTALL 文档**：移除错误的自安装命令，简化为 Clone/Submodule。
2.  **更新 README**：同步简化安装部分，并更新 Step 0 的描述。
3.  **更新 USAGE**：确保详细指南中的 Step 0 逻辑与 README 一致。
4.  **验证**：检查所有修改后的文档，确保路径 `aics-docs/` 的使用在逻辑上是通顺的（作为项目根目录下的一个文件夹）。
