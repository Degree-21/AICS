# 支持“Skills”模式导入文档的计划

用户建议非常好，我们可以将 AICS 的核心能力封装为“Skill（技能）”的形式。这通常意味着提供一个结构化的提示词包（Prompt Package），用户可以将其导入到 AI 助手的技能库中，从而通过简单的指令（如 `/aics-init` 或 `@AICS`）来激活整个工作流，而无需每次手动复制粘贴系统消息。

虽然不同的 AI 工具（如 Trae, Cursor, Windsurf）对“Skill”的定义格式不同，但核心都是“系统提示词（System Prompt）”+“上下文知识（Context）”。

## 1. 新增 Skills 定义文件
我将在 `docs/skills/` 目录下创建通用的技能定义文件，方便用户直接复制使用。

-   **文件位置**：`docs/skills/aics-skill.en.md` 和 `docs/skills/aics-skill.zh.md`
-   **内容包含**：
    -   **Skill Name**: AICS-Architect
    -   **Description**: 规格驱动开发助手 (Spec-Driven Development Assistant)
    -   **System Prompt**: 整合了 CARE 框架、模板路径引用、Step 0-4 工作流的完整指令。
    -   **Usage**: 如何触发（例如“分析项目”、“生成规格”）。

## 2. 更新使用文档
在 `USAGE` 文档中增加“进阶：作为 Skill 导入”章节。

-   **修改文件**：`docs/USAGE.zh.md`, `docs/USAGE.en.md`
-   **新增内容**：
    -   引导用户创建一个名为 "AICS" 的 Skill。
    -   将 `docs/skills/aics-skill.zh.md` 的内容复制为 Skill 的提示词。
    -   （关键）设置 Skill 的 Context 范围为 `aics-docs/` 目录，这样 Skill 就能自动读取模板。

## 3. 优化 Step 0 体验
如果用户使用了 Skill，Step 0（项目认知）可以被自动化。
-   在 Skill 的提示词中预埋指令：*“当用户首次激活此 Skill 时，自动执行 Step 0：扫描根目录并生成 `aics-docs/project-context.md`。”*

## 执行步骤
1.  创建 `docs/skills/` 目录。
2.  编写中英文 Skill 定义文档。
3.  更新 `USAGE` 文档，添加 Skill 导入指南。
4.  更新 `README`，简要提及这种高级用法。
